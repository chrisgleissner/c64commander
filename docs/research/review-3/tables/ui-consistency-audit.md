# UI Consistency and Small-Screen Audit

| Area                               | Observation                                                                                       | Evidence                                                    | Impact                              | Follow-up (small-scope)                                                       |
| ---------------------------------- | ------------------------------------------------------------------------------------------------- | ----------------------------------------------------------- | ----------------------------------- | ----------------------------------------------------------------------------- |
| Bottom tab readability             | Tab labels use `text-[10px]`; six tabs increase crowding risk on 5.5-inch class screens           | `src/components/TabBar.tsx`, `src/index.css` (`.tab-item`)  | Medium readability risk             | Raise label size slightly and enforce truncation/spacing guardrails.          |
| Touch target consistency           | `.tab-item` style does not explicitly enforce 44x44 minimum target                                | `src/index.css` (`.tab-item`)                               | Medium accessibility/usability risk | Add min-height/min-width touch target tokens for nav items.                   |
| Long-text overflow defenses        | Project has explicit overflow/layout tests for 360x740 and dialog boundaries                      | `playwright/layoutOverflow.spec.ts`                         | Positive signal                     | Keep tests mandatory in CI for mobile projects.                               |
| Source-language consistency        | UX guideline enforces intent-based terms and source ordering; tests exist for wording             | `docs/ux-guidelines.md`, `playwright/uxInteractions.spec.ts` | Positive signal                     | Add one cross-page wording snapshot test for regression prevention.           |
| Loading/error behavior consistency | Diagnostics and error surfaces are present, but large page components increase inconsistency risk | `src/pages/HomePage.tsx`, `src/pages/SettingsPage.tsx`      | Medium                              | Add a compact checklist for loading/empty/error visual states per major page. |
