## Cross-cutting observations

- Many of the "degraded" reports trace back to the contributor windows ignoring the deviceId dimension. Fixing F-DIAG-1 (D-1) is the single highest-leverage change.
- A second cluster of issues (D-5/D-6) is about the discovery state machine over-claiming OFFLINE/Not-available during the first 10–15 s of life, even when network traffic is healthy.
- Transport-layer overhead (D-7/D-8) compounds D-5: more requests through CapacitorHttp+CapacitorCookies + per-item fan-out = more time spent in DISCOVERING-equivalent state.

## What this matrix is NOT

- It is **not** a list of "things to mute". Each row is a real cause that produces user-visible incorrect or slow behavior. None of the proposed fixes are label/threshold loosening; they are corrections to data flow, scheduling, or transport.
